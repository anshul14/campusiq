# Contributing to CampusIQ

Thank you for your interest in contributing to CampusIQ.

## Ways to Contribute

- Build a new CMS plugin or enhancing the existing ones.
- Build a new experience plugin or enhance the existing ones.
- Report bugs via GitHub Issues
- Improve documentation
- Bugs fix or new feature implementations

## Getting Started

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Commit with a descriptive message
5. Push to your fork and open a Pull Request

## Plugin Contributions

If you are building a new CMS plugin or experience plugin, 
please refer to the template in:
- `src/application/plugins/content_plugin_interface/template/content_plugin_template.py`
- `src/application/plugins/experience_plugin_interface/experience_plugin_template.py`

## Code Style

- Follow PEP 8 for Python code
- Add docstrings to all classes and public methods
- Include the Apache 2.0 license header in every new file

## Reporting Issues

Please use GitHub Issues and select the appropriate template — 
bug report or feature request.

## Code of Conduct

Be respectful. Be constructive. Focus on the work.
